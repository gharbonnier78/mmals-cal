# MMALS-CAL Positioning: Standalone in Exposition, Companion in Provenance

The MMALS Chronicle (Paper 1) describes the learning architecture: inferred context, route selection, specialized hosts, reconstructive audit memory, and compiled functional memory. MMALS-CAL addresses a different question that appears after prediction:

> Is the evidence associated with the current inferred regime still valid, sufficiently compact, and adequate for action?

MMALS-CAL is therefore a companion to the Chronicle in architectural provenance, but the present repository is standalone in exposition and execution. A reader does not need the Chronicle to reproduce the seven-benchmark calibration analysis.

## Scientific Verification Stack versus operational evidence verification

- **Metric Evaluator:** computes performance and uncertainty metrics.
- **Protocol Verifier:** checks separation, provenance, deterministic selection, and comparable units.
- **MMALS-CAL:** tests calibration validity, regime recall, compactness, and ACTION/NO-DECISION behavior.
- **Claim Verifier:** checks whether the written conclusion is supported, unsupported, or prohibited.

The Verification Stack qualifies the scientific work. MMALS-CAL additionally prepares the logic required by an eventual runtime evidence controller. Version 0.3.2 remains an offline replay, not the runtime controller itself.
